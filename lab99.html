Lab 99: Creating and Using Ansible Roles

« »
Lab Index
  • Scenario
  • Objective
  • What You'll Practice
  • Walkthrough
  • Reference

Convert a growing playbook into a reusable Ansible role with tasks, handlers, and templates. You’ll deploy a simple Nginx site using a production-style structure that scales cleanly as your automation grows.

ansible roles handlers templates ansible-lint

Scenario

You’re managing a growing set of Ansible playbooks and the structure is starting to sprawl. To keep things clean and reusable, you’ll migrate to roles and implement a production-grade pattern: tasks install and configure, handlers restart services only when needed, and templates render host-specific content.

Operator context

Roles are the “unit of reuse” in Ansible. The goal is a layout you can drop into any repo, compose into a site playbook, and lint for consistency.

Objective

  • Create a role skeleton with ansible-galaxy init.
  • Define tasks to install Nginx and deploy a templated landing page.
  • Create a handler that restarts Nginx only when notified.
  • Write a site playbook that applies the role to the web group.
  • Validate structure and syntax with ansible-lint.

What You’ll Practice

  • Role scaffolding: ansible-galaxy init
  • Role anatomy: tasks/, handlers/, templates/, defaults/
  • Template rendering: template: with facts like ansible_hostname
  • Change-driven restarts: notify: + handlers
  • Quality gates: ansible-lint

Walkthrough

Step 1: Create a new role named nginx.
Command 
ansible-galaxy init nginx

This creates the standard role layout. You’ll primarily work in tasks/main.yml, handlers/main.yml, and templates/.

Step 2: Define a task to install Nginx.
Command 
nano nginx/tasks/main.yml
# or
vim nginx/tasks/main.yml
nginx/tasks/main.yml (baseline) 
- name: Install nginx
  package:
    name: nginx
    state: present

The package module keeps this role portable across distros.

Step 3: Add a handler to restart Nginx.
Command 
nano nginx/handlers/main.yml
# or
vim nginx/handlers/main.yml
nginx/handlers/main.yml 
- name: restart nginx
  service:
    name: nginx
    state: restarted

Handlers run only when notified, which prevents unnecessary restarts.

Step 4: Create the HTML template.
Command 
nano nginx/templates/index.html.j2
# or
vim nginx/templates/index.html.j2
nginx/templates/index.html.j2 
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Ansible Role Deployed</title>
  </head>
  <body>
    <h1>Deployed by Ansible</h1>
    <p>Host: {{ ansible_hostname }}</p>
  </body>
</html>

This uses ansible_hostname from gathered facts to render host-specific output.

Step 5: Deploy the template from the role and notify the handler.
Command 
nano nginx/tasks/main.yml
# or
vim nginx/tasks/main.yml
nginx/tasks/main.yml (final) 
- name: Install nginx
  package:
    name: nginx
    state: present

- name: Ensure nginx is enabled and running
  service:
    name: nginx
    state: started
    enabled: yes

- name: Deploy custom index.html
  template:
    src: index.html.j2
    dest: /usr/share/nginx/html/index.html
    owner: root
    group: root
    mode: "0644"
  notify: restart nginx

The handler only runs if the template task reports a change.

Step 6: Create a site playbook that uses the role.
Command 
nano site.yml
# or
vim site.yml
site.yml 
- name: Configure web servers
  hosts: web
  become: yes
  roles:
    - nginx

This composes the role into a clean entry point. As you add roles, you keep site.yml readable and predictable.

Step 7: Run the role-based playbook.
Command 
ansible-playbook -i ~/inventory site.yml

First run should show changes; subsequent runs should converge to mostly ok unless you modify templates or tasks.

Step 8: Lint the playbook (and role) for best practices.
Command 
ansible-lint site.yml

Linting is your “production hygiene” step: consistent patterns, fewer surprises, and cleaner reviews when roles become shared artifacts across teams.

Reference

  • Role scaffolding
    • ansible-galaxy init nginx
  • Core role files
    • nginx/tasks/main.yml
    • nginx/handlers/main.yml

  • Deploy

  • Quality gate
    • ansible-lint site.yml
