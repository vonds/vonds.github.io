Lab 111: Proc CPU & Memory Exploration

« »
Back to Lab Index

Extract CPU and memory facts directly from /proc and validate them with standard tooling. Build operational confidence by cross-checking raw kernel-exposed data with lscpu and free.

troubleshooting performance core

Scenario

A teammate claims a host is “under-provisioned” and insists it has fewer CPU threads and less memory than expected. You need to validate the facts from the kernel’s source of truth and confirm them using standard utilities that others on the team will recognize.

Operator context

/proc is the raw interface to kernel state. Use it to verify claims quickly, then cross-check with higher-level tooling so your results are portable across teams, tickets, and incident notes.

Objective

  • Extract CPU model information from /proc/cpuinfo.
  • Determine logical CPU count from /proc/cpuinfo.
  • Cross-check CPU facts with lscpu.
  • Read MemTotal and MemAvailable from /proc/meminfo.
  • Cross-check memory results using free -h.
  • Inspect hugepage configuration when present.

What You’ll Practice

  • CPU identification via /proc/cpuinfo and safe extraction patterns using grep and awk .
  • Logical CPU counting and how it maps to lscpu output.
  • Memory inspection via /proc/meminfo and the meaning of “available” memory on Linux.
  • Cross-check methodology for repeatable verification.
  • Hugepages visibility for performance and virtualization contexts.

Walkthrough

Step 1 : Print the CPU model name from /proc/cpuinfo.
Command 
grep -m1 '^model name' /proc/cpuinfo

/proc/cpuinfo exposes CPU details reported by the kernel. Grabbing the first occurrence is usually sufficient because the model name repeats for each logical processor. 

# Expected output format:
model name  : Intel(R) ...
Step 2 : Count logical processors from /proc/cpuinfo.
Command 
grep -c '^processor' /proc/cpuinfo

Each processor stanza represents one logical CPU as seen by the kernel. This count should match CPU(s) reported by lscpu. 

# Example:
8
Step 3 : Cross-check CPU facts with lscpu.
Command 
lscpu

lscpu summarizes CPU topology (threads per core, cores per socket, sockets). Use it to translate raw kernel data into the vocabulary your team expects in tickets and capacity discussions. 

# Key fields to compare:
CPU(s):                  8
Thread(s) per core:      2
Core(s) per socket:      4
Model name:              Intel(R) ...
Step 4 : Read memory totals from /proc/meminfo.
Command 
grep -E '^(MemTotal|MemAvailable)' /proc/meminfo

MemTotal is total physical RAM detected by the kernel. MemAvailable is a best-effort estimate of memory available for new workloads without swapping, accounting for reclaimable cache. 

# Example:
MemTotal:       ... kB
MemAvailable:   ... kB
Step 5 : Cross-check memory with free -h.
Command 
free -h

free presents memory in a format most admins expect. Compare the total and available lines to confirm they align with the values you pulled from /proc/meminfo. 

# Focus on:
total
available
Step 6 : Inspect hugepage settings when present.
Command 
grep -E '^Huge' /proc/meminfo

Hugepages are relevant in performance and virtualization environments. Even when disabled, this view confirms the current size and whether any hugepages are reserved. 

# Example:
HugePages_Total:       0
HugePages_Free:        0
Hugepagesize:       2048 kB

Reference

  • /proc/cpuinfo : Kernel-exposed CPU information (per-logical CPU details).
  • /proc/meminfo : Kernel-exposed memory accounting (totals, available, caches, hugepages).
  • lscpu : Human-readable CPU topology summary (threads, cores, sockets, model).
  • free -h : Memory summary in human-readable units.
    • -h: Scales units automatically.
  • grep and awk : Safe extraction tools for pulling specific fields without manual scanning.
