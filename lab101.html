Lab 101: Secure Secrets with Ansible Vault

« »
Lab Index
  • Scenario
  • Walkthrough
  • Reference

Encrypt secrets at rest using Ansible Vault so credentials can be version-controlled safely. Practice the full operator workflow: create, view, edit, use in a playbook run, and rotate vault passwords.

automation security ansible

Scenario

Your Ansible repo needs to store database credentials and API tokens, but storing secrets in plaintext is not acceptable. You will use ansible-vault to encrypt secrets at rest, load them during a playbook run, and rotate the vault password.

Operator context

Vault is the baseline for “secrets in Git” workflows. It reduces accidental leakage, supports reviews, and enables controlled rotation without rewriting your entire automation stack.

Objective

  • Create a Vault-encrypted secrets file using ansible-vault create .
  • Validate ciphertext on disk and safely edit/view contents.
  • Load the vault file via vars_files in a playbook.
  • Run a playbook using --ask-vault-pass .
  • Rotate the vault password using ansible-vault rekey .

What You’ll Practice

  • Vault file lifecycle: create , view , edit , rekey .
  • Vault header recognition and “ciphertext vs plaintext” sanity checks.
  • Playbook integration using vars_files and safe output via debug .
  • Password rotation as a routine operational control.

Walkthrough

Step 1 : Create a new Vault-encrypted file for credentials.
Command 
ansible-vault create secrets.yml

This command prompts for a vault password and opens an editor. The file is encrypted on save. While editing, the contents are plaintext in your editor buffer only. 

  New Vault password:
  Confirm New Vault password:
    [opens editor: /usr/bin/vi]

    --- secrets.yml (plaintext while editing, then encrypted on save) ---
    db_user: appuser
    db_pass: 'S3cure!ChangeMe'
    db_host: 10.10.20.15
    -------------------------------------------------------------------

  Encryption successful
Step 2 : Validate that the file is ciphertext on disk.
Command 
cat secrets.yml

A vault file begins with the vault header. If you see plaintext here, you are not encrypted.

Step 3 : Edit the vault file securely.
Command 
ansible-vault edit secrets.yml

Vault prompts for the password, decrypts in memory, opens the editor, then re-encrypts on save. 

  Vault password:
    [opens editor: /usr/bin/vi]
  Encryption successful
Step 4 : View decrypted contents (plaintext to stdout).
Command 
ansible-vault view secrets.yml

This prints plaintext to your terminal. In real operations, avoid running this in shared terminals or in recorded sessions. 

  Vault password:
  db_user: appuser
  db_pass: 'S3cure!ChangeMe'
  db_host: 10.10.20.15
Step 5 : Create a playbook that loads secrets.yml via vars_files.
File 
# vault-playbook.yml
- name: Test vault secrets
  hosts: localhost
  gather_facts: false
  vars_files:
    - secrets.yml
  tasks:
    - name: Show DB connection target
      debug:
        msg: "DB={{ db_user }}@{{ db_host }}"

vars_files loads variables from your Vault file, and Ansible decrypts it during the run when provided the vault password.

Step 6 : Run the playbook using --ask-vault-pass.
Command 
ansible-playbook vault-playbook.yml --ask-vault-pass
  Vault password:

  PLAY [Test vault secrets] ******************************************************

  TASK [Show DB connection target] ***********************************************
  ok: [localhost] => {
      "msg": "DB=appuser@10.10.20.15"
  }

  PLAY RECAP *********************************************************************
  localhost                  : ok=1    changed=0    unreachable=0    failed=0    skipped=0    rescued=0    ignored=0
Step 7 : Rotate the vault password (rekey).
Command 
ansible-vault rekey secrets.yml
  Vault password:
  New Vault password:
  Confirm New Vault password:
  Rekey successful

Rekey changes the password used to decrypt the vault file. In real teams, this is part of routine credential rotation.

Reference

  • ansible-vault create <file> : Create a new encrypted file (prompts for password, opens editor).
  • ansible-vault edit <file> : Decrypts in memory, opens editor, re-encrypts on save.
  • ansible-vault view <file> : Prints decrypted content to stdout.
  • ansible-vault rekey <file> : Rotates the vault password for the file.
  • ansible-playbook --ask-vault-pass : Prompts for vault password during a playbook run.
  • vars_files: : Loads variables from external YAML files (including vault files).
