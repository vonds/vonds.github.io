Lab 108: Kernel Module Fundamentals

« »
Lab Index

  • Walkthrough
  • Reference

Inspect loaded kernel modules, read module metadata, and practice loading and unloading a safe module using standard tools. Verify changes using module listing and kernel-exposed module state.

kernel modules core

Scenario

Your team needs a repeatable workflow for inspecting kernel module state during troubleshooting. You will list loaded modules, inspect module metadata, load and unload a safe module, and verify state changes using standard tools.

Operator context

Always treat module operations as a controlled change. In production, unloading the wrong module can impact storage, networking, graphics, or security controls.

Objective

  • View currently loaded kernel modules and interpret the output header.
  • Inspect a module’s metadata using modinfo.
  • Load a safe module using modprobe.
  • Verify the module is present using lsmod or /proc/modules.
  • Practice passing a module parameter during load.
  • Remove the module safely and confirm removal.

What You’ll Practice

  • Module state inspection with lsmod and basic output filtering.
  • Module metadata inspection with modinfo (file path, description, license, parameters, aliases).
  • Safe module load and unload workflows with modprobe and modprobe -r .
  • Verifying kernel state via /proc/modules as an alternate source of truth.
  • Parameter passing patterns (what to expect and when it applies).

Walkthrough

Step 1 : Show the first 10 lines of loaded kernel modules.
Command 
lsmod | head -n 10

lsmod shows what is loaded right now. The “Used by” count helps you estimate whether a module is likely to unload cleanly, and what other modules may be depending on it. 

Module                  Size  Used by
xfs                   999424  1
e1000                 155648  0
Step 2 : Display metadata for the loop module.
Command 
modinfo loop

modinfo answers “what is this module,” “where is it on disk,” and “what options does it accept.” Use this to confirm you are operating on the correct driver before loading it or troubleshooting why it will not load. 

filename:       /lib/modules/$(uname -r)/kernel/drivers/block/loop.ko
description:    Loopback block device
license:        GPL
parm:           max_loop:int
Step 3 : Load the loop module into the kernel.
Command 
sudo modprobe loop

modprobe loads the module and resolves dependencies automatically. If the module is already loaded or built-in, this may produce no output. 

# No output is normal on success.
# Confirm in the next step.
Step 4 : Confirm that loop appears in the list of loaded modules.
Command 
lsmod | grep -E '^loop(\s|$)'

Filtering lsmod output is the fastest way to prove the module is present. If the module does not appear, check whether it is built into the kernel or whether the load failed. 

loop                   45056  0
Alternate 
grep loop /proc/modules

/proc/modules is a kernel-exported view of loaded modules and can be used as an alternate verification source.

Step 5 : Reload or ensure loop with a parameter setting.
Command 
sudo modprobe loop max_loop=8

Module parameters are applied at load time. If the module is already loaded, parameter changes may not take effect until it is unloaded and loaded again. 

# If you need to apply a parameter reliably:
# sudo modprobe -r loop && sudo modprobe loop max_loop=8
Step 6 : Remove the loop module from the kernel.
Safety note

A module may refuse to unload if it is in use. That failure is a safety feature, not a bug. Always understand what depends on the module before forcing removal.

Command 
sudo modprobe -r loop

modprobe -r removes the module and updates kernel module state. If the module is still needed, the command may fail with an “in use” message. 

# Verify removal:
lsmod | grep -E '^loop(\s|$)'
Alternate 
sudo rmmod loop

rmmod removes a module directly, but it does not resolve dependencies. Prefer modprobe -r for normal operations.

Reference

  • lsmod : Lists currently loaded kernel modules.
    • Output includes module name, size, and “Used by” count.
  • modinfo <module> : Displays metadata for a kernel module (file path, description, license, parameters, aliases).
  • modprobe <module> [param=value] : Loads a module and resolves dependencies.
    • Parameters are applied at load time.
  • modprobe -r <module> : Unloads a module with dependency awareness.
    • Will refuse to unload if the module is in use.
  • rmmod <module> : Removes a module directly without dependency resolution.
  • /proc/modules : Kernel-exported view of loaded modules.
