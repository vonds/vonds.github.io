Lab 84: Setting Up a Proxy Server with Squid

« »
Lab Index
  • Scenario
  • Walkthrough
  • Reference

Install and bring up Squid as a basic HTTP proxy service. Verify the service is enabled and confirm it is listening on port 3128.

networking proxy services

Scenario

Install and configure Squid as a basic proxy server for HTTP traffic. Your task is to install the package, enable the service, and validate that the proxy port is reachable.

Operator context

This is a standard “bring the service online” workflow: install, enable, verify listener state, and confirm the port is open before you introduce ACL restrictions, authentication, or TLS interception.

Objective

  • Install Squid using your distro’s package manager.
  • Enable and start the Squid service with systemd.
  • Confirm Squid is listening on the default proxy port 3128 .

What You’ll Practice

  • Installing network services via apt , dnf , or pacman .
  • Enabling and starting a service with systemctl enable --now .
  • Validating listener state with ss -tuln .
  • Understanding the default Squid proxy port ( 3128 ).

Walkthrough

Step 1 : Install the Squid proxy package.
Command 
# Pick the command that matches your distro:

# Debian/Ubuntu
sudo apt install squid -y

# RHEL/Fedora
sudo dnf install squid -y

# Arch
sudo pacman -S squid

Squid provides a forward proxy service that can mediate outbound web requests for clients. Installation lays down the default configuration and the systemd unit required to start the service. 

# Example result (varies by distro):
# squid installed successfully.
Step 2 : Enable and start Squid.
Command 
sudo systemctl enable --now squid

This starts Squid immediately and ensures it will start at boot. If it fails, check systemctl status squid and review logs for config or permission problems. 

# Quick verification:
systemctl status squid
Step 3 : Confirm the proxy port is open (default 3128).
Command 
ss -tuln | grep 3128

This confirms Squid has successfully bound to its listening port and is ready to accept proxy connections. If nothing is listening, the service may not be running, or Squid may be configured to listen on a different port. 

LISTEN 0      100        0.0.0.0:3128       0.0.0.0:*

Reference

  • squid : Forward proxy server commonly used for web caching and access control.
  • apt install squid -y : Installs Squid on Debian/Ubuntu.
  • dnf install squid -y : Installs Squid on Fedora/RHEL-family systems.
  • pacman -S squid : Installs Squid on Arch-based systems.
  • systemctl enable --now squid : Enables Squid at boot and starts it immediately.
  • ss -tuln : Lists listening TCP/UDP sockets without name resolution.
  • 3128 : Default Squid proxy port (unless overridden in config).
