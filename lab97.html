Lab 97: Set up a basic NTP client with chronyd

« »
Lab Index
  • Scenario
  • Objective
  • What You'll Practice
  • Walkthrough
  • Reference

Configure a host as an NTP client using chronyd so logs and timestamps stay consistent across systems. Verify sync health with timedatectl and chronyc, then apply a one-time step correction when drift is large.

chrony ntp time-sync rhel

Scenario

A server’s clock is drifting and logs are no longer ordered correctly. Your job is to install chrony, enable chronyd, verify that the system is synchronized, and confirm the active NTP sources and tracking statistics.

Operator context

Time drift causes subtle failures: TLS issues, confusing incident timelines, broken log correlation, and unpredictable authentication behavior. The goal is “enable time sync, prove it’s working, then document what it’s using.”

Objective

  • Install chrony and enable chronyd.
  • Verify service health with systemctl.
  • Confirm OS time sync status with timedatectl.
  • Validate active NTP sources with chronyc sources -v.
  • Apply a one-time step correction using chronyc makestep.
  • Review tracking stats and confirm config file location.

What You’ll Practice

  • Service control and verification: systemctl enable --now, systemctl status --no-pager
  • Time sync verification: timedatectl status
  • Chrony admin tooling: chronyc sources -v, chronyc tracking, chronyc makestep
  • Basic config awareness: /etc/chrony.conf

Walkthrough

Step 1: Install chrony.
Command 
sudo dnf install chrony -y
# (older systems may use yum)
# sudo yum install chrony -y

Once installed, treat time sync as “baseline infrastructure” the same way you treat logging and DNS: enable it early and verify it often.

Step 2: Enable and start chronyd.
Command 
sudo systemctl enable --now chronyd

This activates sync immediately and ensures it persists across reboots.

Step 3: Check service status.
Command 
systemctl status chronyd --no-pager

You’re confirming “active (running)” and scanning for obvious errors before moving on to source and tracking validation.

Step 4: Verify sync status via timedatectl.
Command 
timedatectl status

Look for signals like System clock synchronized: yes and an active NTP service.

Step 5: Check NTP sources.
Command 
chronyc sources -v

The selected source is typically marked with *. You’re verifying that the host can reach time sources and that at least one is healthy.

Step 6: Force a one-time step correction.
Command 
sudo chronyc makestep

This is appropriate when drift is large and you need timestamps corrected quickly. Afterward, you still verify tracking and source state.

Step 7: Review tracking statistics.
Command 
chronyc tracking

This is where you confirm the current offset, frequency correction, stratum, and the reference source being used.

Step 8: Confirm config file location.
Command 
ls -l /etc/chrony.conf

This is the primary place you’d document or adjust servers/pools, access controls, and any local policy decisions.

Reference

  • Install + service
    • sudo dnf install chrony -y
    • sudo systemctl enable --now chronyd
    • systemctl status chronyd --no-pager
  • Verify sync
    • timedatectl status
    • chronyc sources -v
    • chronyc tracking
    • sudo chronyc makestep
  • Config
    • /etc/chrony.conf
