Lab 121: Configure DNF Repositories (RHEL)

« »
Lab Index
  • Scenario
  • Walkthrough
  • Reference

Audit enabled repositories, inspect repo definitions under /etc/yum.repos.d , add a new .repo file safely, validate metadata with dnf makecache , then disable a broken repo cleanly without deleting evidence.

packages repos troubleshooting rhel

Scenario

A RHEL-like server was cloned into a lab network and package installs are failing. The repo configuration may be incomplete, or a custom repo was added incorrectly. Your job is to audit what is enabled, verify repo definitions on disk, add a repo file safely, validate metadata, then disable the broken repo cleanly.

Operator context

Repo failures can block patching and incident response. Treat repo files as configuration artifacts: back up before changes, validate repodata, and disable cleanly so you can restore service without losing context.

Objective

  • List enabled repositories and inspect a repoâ€™s details.
  • Audit repo definition files under /etc/yum.repos.d.
  • Back up an existing repo file before editing.
  • Create a new .repo file with a valid repo id.
  • Validate repodata and caching behavior with dnf makecache.
  • Disable a broken repo without deleting the file.
  • Clean metadata and verify the repo is no longer enabled.

Walkthrough

Step 1 : List enabled repositories.
Command 
dnf repolist enabled

This is the fastest baseline check when installs fail. If the expected repos are missing or disabled, nothing downstream will work. 

repo id                      repo name
appstream                    Rocky Linux 9 - AppStream
baseos                       Rocky Linux 9 - BaseOS
extras                       Rocky Linux 9 - Extras
Step 2 : Show repo definitions currently present on disk.
Command 
ls -1 /etc/yum.repos.d

Repos are defined by .repo files in this directory. Missing files often explains missing repos. 

rocky.repo
rocky-addons.repo
Step 3 : Back up an existing repo file before changes.
Command 
sudo cp /etc/yum.repos.d/rocky.repo /etc/yum.repos.d/rocky.repo.bak

This is the safety net. If a repo change breaks installs, you can restore quickly. 

Backup created: /etc/yum.repos.d/rocky.repo.bak
Step 4 : Inspect details for one enabled repo (BaseOS).
Command 
dnf repoinfo baseos

Confirm the baseurl and whether the repo is enabled. A bad URL usually turns into metadata errors. 

Repo-id            : baseos
Repo-name          : Rocky Linux 9 - BaseOS
Repo-baseurl       : https://download.rockylinux.org/pub/rocky/9/BaseOS/x86_64/os/
Repo-enabled       : yes
Step 5 : Add a custom repo file under /etc/yum.repos.d ( custom.repo ).
Command 
sudo vim /etc/yum.repos.d/custom.repo

A repo file must contain at least a repo id in brackets and a baseurl . This lab uses a deliberately broken URL so you can recognize the failure mode. 

[custom-tools]
name=Custom Tools Repo
baseurl=https://repo.example.lab/rhel/9/x86_64/
enabled=1
gpgcheck=0
Step 6 : Verify the repo file is readable and contains a repo id.
Command 
sudo cat /etc/yum.repos.d/custom.repo

This is a quick integrity check. You want to see the bracketed repo id and the URL. Typos here show up later as metadata failures.

Step 7 : Build metadata cache (validate DNF can read repo configuration).
Command 
sudo dnf makecache

This validates the repo file and attempts to fetch repodata/repomd.xml . A 404 usually means the baseurl is wrong or the repo is not structured correctly. 

Custom Tools Repo                                0.0  B/s |   0  B     00:00
Errors during downloading metadata for repository 'custom-tools':
  - Status code: 404 for https://repo.example.lab/rhel/9/x86_64/repodata/repomd.xml
Step 8 : Disable the custom repo safely (without deleting the file).
Command 
sudo dnf config-manager --set-disabled custom-tools

Disabling keeps the configuration on disk for review, while immediately unblocking package operations. 

Repository 'custom-tools' is disabled.
Step 9 : Clean metadata after disabling the repo.
Command 
sudo dnf clean metadata

This removes cached metadata so you can rebuild cleanly and confirm the error no longer appears. 

0 files removed
Step 10 : Confirm the custom repo no longer appears as enabled.
Command 
dnf repolist enabled

Verification is the final step. You should see only expected repos enabled. 

repo id                      repo name
appstream                    Rocky Linux 9 - AppStream
baseos                       Rocky Linux 9 - BaseOS
extras                       Rocky Linux 9 - Extras
(custom-tools is disabled)

Reference

  • /etc/yum.repos.d/*.repo : Repository definitions (repo id, baseurl, enabled, gpgcheck).
  • dnf repolist enabled : Baseline check for enabled repos.
  • dnf repoinfo <repoid> : Inspect baseurl and enabled state.
  • dnf makecache : Validate repo config and fetch repodata.
  • dnf config-manager --set-disabled <repoid> : Disable without deleting repo files.
  • dnf clean metadata : Remove cached metadata after repo changes.
  • repodata/repomd.xml : Core metadata file; missing usually means baseurl or repo layout is wrong.
