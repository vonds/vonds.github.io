Lab 88: Managing Log Rotation with logrotate

« »
Back to Lab Index

  • Walkthrough
  • Reference

Manage log growth and retention using logrotate by defining a custom rotation policy for a sample application log. Force a rotation run, validate rotated artifacts, and review global defaults.

logging storage operations

Scenario

You need to manage log growth and retention using the logrotate utility. Your task is to create a custom rotation policy for a sample log file and apply it manually to validate rotation and compression behavior.

Operator context

Uncontrolled log growth is a common cause of disk pressure. A clean rotation policy reduces risk while preserving evidence for incident response and troubleshooting.

Objective

  • Ensure the logrotate package is installed.
  • Create a test log file under /var/log.
  • Define a custom rotation policy in /etc/logrotate.d .
  • Force a rotation run with logrotate -f .
  • Confirm rotated output exists and is compressed as expected.
  • Review global defaults in /etc/logrotate.conf .

What You’ll Practice

  • Installing and validating core tooling via apt , dnf , or pacman .
  • Creating a drop-in logrotate policy under /etc/logrotate.d .
  • Interpreting common directives: weekly , rotate , compress , missingok , notifempty .
  • Forcing a controlled rotation run to validate behavior.
  • Verifying output files and understanding retention outcomes.

Walkthrough

Step 1 : Ensure logrotate is installed.
Command 
# Pick the command that matches your distro:

# Debian/Ubuntu
sudo apt install logrotate -y

# RHEL/Fedora
sudo dnf install logrotate -y

# Arch
sudo pacman -S logrotate

logrotate is typically run on a schedule (often daily) to rotate logs according to policy. Installing it ensures the binary, global config, and include directory are present. 

# Example install outcomes (varies by distro):
logrotate is already the newest version.
# or
Nothing to do. Complete!
Step 2 : Create a sample log file to rotate.
Command 
sudo touch /var/log/testapp.log

This creates a log target that mimics an application log under /var/log . You’ll attach a custom policy to this file in the next step. 

# Confirm the file exists:
ls -l /var/log/testapp.log
Step 3 : Create a logrotate policy for testapp.log.
Command 
sudo nano /etc/logrotate.d/testapp
# or
sudo vim /etc/logrotate.d/testapp

Drop-in policies under /etc/logrotate.d keep application-specific rules separated from global defaults. This one rotates weekly, keeps 4 archives, compresses rotated output, ignores missing files, and skips empty logs. 

/var/log/testapp.log {
    weekly
    rotate 4
    compress
    missingok
    notifempty
}
Step 4 : Force a rotation run for the custom policy.
Command 
sudo logrotate -f /etc/logrotate.d/testapp

The -f flag forces rotation regardless of whether the normal time threshold has been met. This is how you validate a policy safely during setup or troubleshooting. 

# Expected outcome:
# Forced rotation for /var/log/testapp.log completed.
Step 5 : Confirm rotated artifacts exist.
Command 
ls /var/log/testapp.log*

You should see the active log and at least one rotated file. With compress enabled, the rotated file is typically gzip-compressed ( .gz ). 

/var/log/testapp.log
/var/log/testapp.log.1.gz
Step 6 : Review global defaults in /etc/logrotate.conf.
Command 
cat /etc/logrotate.conf

The global configuration sets baseline defaults and commonly includes /etc/logrotate.d for per-app rules. Understanding global defaults helps you avoid surprises when troubleshooting rotation behavior. 

# see 'man logrotate' for details
# global options for log rotation
weekly
rotate 4
create
include /etc/logrotate.d
# system-specific logs may be listed here

Reference

  • logrotate : Utility that rotates, compresses, and retires log files according to policy.
  • /etc/logrotate.conf : Global defaults for logrotate behavior (often includes /etc/logrotate.d ).
  • /etc/logrotate.d/ : Directory for per-application rotation policies.
  • logrotate -f <config> : Forces a rotation run using the specified config file.
  • weekly : Rotation frequency directive.
  • rotate 4 : Retain 4 rotated archives before removing the oldest.
  • compress : Compress rotated logs (typically gzip).
  • missingok : Do not error if the log file is missing.
  • notifempty : Do not rotate empty logs.
