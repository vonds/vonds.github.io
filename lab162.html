Deploy Obsidian on a managed workstation using Flatpak when RPM installs are not allowed. Verify remotes, install, validate, update, and uninstall using standard Flatpak workflows.

flatpak desktop support

Scenario

A developer on a managed workstation requests Obsidian for documentation. They cannot install RPMs due to policy controls. You must deploy the app using Flatpak, verify that Flatpak is functional, add Flathub if missing, install the app, confirm it runs, apply updates, and remove it cleanly when requested.

Operator context

In locked-down environments, app delivery often shifts from package managers to sandboxed runtimes. Flatpak is a standard path to ship desktop apps without breaking workstation policy.

Objective

  • Confirm Flatpak is installed and functional.
  • Review configured remotes and add Flathub if missing.
  • Search for Obsidian and identify the correct application ID.
  • Install Obsidian, verify installation details, and confirm it launches.
  • Update Flatpak applications and uninstall Obsidian when requested.

Concepts

  • Flatpak as a policy-friendly deployment path for managed workstations.
  • Remotes (repositories) and system-wide versus user installations.
  • Application IDs as the stable install/run/uninstall identifier.
  • Verifying state with flatpak list and flatpak info.
  • Operational hygiene: update regularly and remove apps cleanly when requested.

Walkthrough

Step 1 : Confirm Flatpak is installed.
Command 
flatpak --version
  Flatpak 1.14.6

Confirming the version is a fast signal that the client is present and runnable. If Flatpak is missing, the request becomes a policy or base-image problem rather than an application problem.

Step 2 : Check configured Flatpak remotes.
Command 
flatpak remotes --show-details
  Name    Title   URL                                      Collection ID  Subset  Filter  Priority  Options
  fedora  Fedora  oci+https://registry.fedoraproject.org    -              -       -       1         system

Remotes determine where apps and runtimes come from. Many managed systems ship with a vendor remote but not Flathub.

Step 3 : Add Flathub system-wide if it is missing.
Change control

Adding a new remote changes where software can be sourced. In enterprise environments, this is often ticketed and approved explicitly.

Command 
sudo flatpak remote-add --if-not-exists flathub https://flathub.org/repo/flathub.flatpakrepo
  [sudo] password for lab:
  Remote 'flathub' successfully added.

--if-not-exists makes this safe to rerun in automation or repeatable lab workflows.

Step 4 : Confirm Flathub appears in the remote list.
Command 
flatpak remotes
  Name
  fedora
  flathub

This confirms your software source is configured before you attempt to search or install.

Step 5 : Search for Obsidian and capture the application ID.
Command 
flatpak search obsidian
  Name      Description                    Application ID         Version  Branch  Remotes
  Obsidian  Markdown-based knowledge base  md.obsidian.Obsidian   1.5.8    stable  flathub

The application ID is what you will use for install, run, info, and uninstall. It is the reliable identifier to document in tickets and runbooks.

Step 6 : Install Obsidian from Flathub.
Command 
flatpak install -y flathub md.obsidian.Obsidian
  Looking for matches…
  Required runtime for md.obsidian.Obsidian/x86_64/stable (runtime/org.freedesktop.Platform/x86_64/23.08) found in remote flathub
  Installing…
  1. [####################] org.freedesktop.Platform 23.08
  2. [####################] md.obsidian.Obsidian 1.5.8 stable
  Installation complete.

Flatpak installs the application plus required runtimes. Expect the first install on a workstation to pull a platform runtime.

Step 7 : Confirm Obsidian is installed.
Command 
flatpak list --app
  Name      Application ID         Version  Branch  Origin   Installation
  Obsidian  md.obsidian.Obsidian   1.5.8    stable  flathub  system

Use this output for quick confirmation and to verify whether the install is system-wide or per-user.

Step 8 : Capture install details for documentation.
Command 
flatpak info md.obsidian.Obsidian
  ID: md.obsidian.Obsidian
  Ref: app/md.obsidian.Obsidian/x86_64/stable
  Arch: x86_64
  Branch: stable
  Version: 1.5.8
  Origin: flathub
  Installation: system
  Runtime: org.freedesktop.Platform/x86_64/23.08

flatpak info provides enough detail to close the ticket cleanly: app ID, origin, branch, version, and runtime.

Step 9 : Launch Obsidian to confirm it starts.
Command 
flatpak run md.obsidian.Obsidian
  [flatpak] Launching md.obsidian.Obsidian...
  (GUI application started)
  (User confirms app opens)
  (Application exited)

The operational success signal is that the user can open the application and use it normally. When you cannot verify via CLI alone, confirm directly with the requester.

Step 10 : Apply Flatpak updates.
Command 
flatpak update -y
  Looking for updates…
  Nothing to do.

Keep runtimes and apps current. Updates are part of workstation hygiene, even if the requested app itself has no new version available.

Step 11 : Uninstall Obsidian.
Command 
flatpak uninstall -y md.obsidian.Obsidian
  Uninstalling md.obsidian.Obsidian…
  Uninstallation complete.

Removal requests are part of normal workstation support. Document that the app was removed and confirm whether runtimes should remain for other Flatpak apps.

Common breakpoints

flatpak command not found

Flatpak is not installed on the workstation. This becomes a baseline image or policy issue, not an app deployment issue.

flathub remote add fails

You may be blocked by policy, missing sudo access, or outbound network restrictions. Confirm approval and verify connectivity to the remote URL.

search shows multiple matches

Prefer the official application ID and stable branch. Document the chosen ID and origin explicitly to avoid installing lookalikes.

app launches but user reports failure

GUI apps can fail due to missing portals, permissions, or graphics stack issues. Capture user-visible errors, then inspect Flatpak permissions and runtime compatibility.

Cleanup checklist

This lab modifies Flatpak remotes and installs an application. If you are resetting a lab host, remove Obsidian and confirm the remote state matches your baseline.

Commands 
flatpak uninstall -y md.obsidian.Obsidian
flatpak remotes

Reference

  • flatpak --version : Prints the installed Flatpak version.
  • flatpak remotes --show-details : Lists configured remotes with detailed metadata.
    • --show-details : Includes URL, priority, and installation scope.
  • sudo flatpak remote-add --if-not-exists flathub <url> : Adds the Flathub remote system-wide if it is not already present.
    • --if-not-exists : Avoids errors if the remote is already configured.
    • <url> : Flatpak repo definition (for example Flathub).
  • flatpak remotes : Lists configured remote names.
  • flatpak search <term> : Searches remotes for matching applications.
  • flatpak install -y flathub md.obsidian.Obsidian : Installs an application from a remote.
    • -y : Automatically answers “yes” to prompts.
    • flathub : Remote name used as the install source.
  • flatpak list --app : Lists installed Flatpak applications.
    • --app : Shows only applications (not runtimes).
  • flatpak info md.obsidian.Obsidian : Displays detailed metadata for an installed application.
  • flatpak run md.obsidian.Obsidian : Launches an installed Flatpak application by ID.
  • flatpak update -y : Updates installed Flatpak apps and runtimes.
    • -y : Automatically answers “yes” to prompts.
  • flatpak uninstall -y md.obsidian.Obsidian : Uninstalls an application by ID.
    • -y : Automatically answers “yes” to prompts.
