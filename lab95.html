Lab 95: Tuning System Performance

« »
Lab Index
  • Scenario
  • Objective
  • What You'll Practice
  • Walkthrough
  • Reference

Apply a tuned profile to align system behavior with workload goals, then control CPU scheduling pressure by adjusting process priorities with nice and renice. Verify active profiles and confirm that your priority changes took effect on running processes.

performance scheduling rhel

Scenario

A host is being repurposed to handle a heavier workload. Your job is to apply a tuned performance profile appropriate for throughput and then reduce the impact of noisy background CPU jobs by controlling process priorities. You’ll verify tuned state, start test workloads, and use nice/renice to adjust scheduling priority safely.

Operator context

tuned changes system behavior at a profile level. nice/renice is a per-process control and won’t “fix” a broken workload, but it can keep low-value jobs from competing with critical services.

Objective

  • Install and start tuned.
  • List available tuned profiles and apply a new profile.
  • Verify the currently active profile.
  • Generate CPU load for testing and compare priorities.
  • Start a process with a lower priority using nice.
  • Change a running process’s nice value using renice.
  • Clean up background workloads.

What You’ll Practice

  • tuned service lifecycle and profile management.
  • tuned-adm commands: list, profile, active.
  • Process priority basics: lower priority (higher nice number) vs higher priority (lower nice number).
  • Verifying priority with ps output (NI column).
  • Safer cleanup patterns: kill by PID, pkill/killall when appropriate.

Walkthrough

Step 1 : Install tuned (if needed).
Command 
# Debian/Ubuntu
sudo apt install tuned -y

# RHEL/Fedora
sudo dnf install tuned -y

# Arch (if available in your repo set)
sudo pacman -S tuned

tuned is most common in RHEL-family environments, but the workflow is consistent: install it, start it, then manage profiles.

Step 2 : Enable and start tuned.
Command 
sudo systemctl enable --now tuned

This turns on profile enforcement and ensures tuned will start on boot.

Step 3 : List available tuned profiles.
Command 
tuned-adm list

You’re looking for profiles like balanced, powersave, throughput-performance, and other workload-specific options depending on distro/version.

Step 4 : Apply the throughput profile.
Command 
sudo tuned-adm profile throughput-performance

This is a common server-oriented profile that tends to favor throughput over power savings. Always verify the active profile after changing it.

Step 5 : Confirm the active profile.
Command 
tuned-adm active

Treat this as a verification step before moving on to process-level tuning.

Step 6 : Start a CPU-intensive background process.
Command 
yes > /dev/null &

This creates a predictable CPU load so you can see how priority changes behave.

Step 7 : Start the same workload with a lower priority using nice.
Command 
nice -n 10 yes > /dev/null &

A higher nice value means lower scheduling priority. This is the “be nice” version of the workload so it competes less aggressively for CPU.

Step 8 : Identify PIDs and inspect nice values.
Command 
# find PIDs
pgrep yes

# inspect priority (NI column shows nice)
ps -o pid,ni,cmd -C yes

Use ps -o pid,ni,cmd -C yes to confirm one process has NI 0 and the nice-started one has NI 10.

Step 9 : Adjust priority of a running process with renice.
Command 
# Example: lower priority (increase nice) of PID 1234
sudo renice 15 -p 1234

# Verify the change
ps -o pid,ni,cmd -p 1234

In practice, you typically increase nice values (lower priority) for background jobs. Decreasing nice values (higher priority) requires elevated privileges and should be used sparingly.

Step 10 : Clean up the test workloads.
Command 
# Either option is fine in a lab environment:
killall yes
# OR
pkill yes

Clean shutdown of test noise matters—especially in a repeatable lab cycle.

Reference

  • tuned-adm
    • tuned-adm list : List profiles and show current selection.
    • tuned-adm profile NAME : Apply a profile.
    • tuned-adm active : Show the active profile.
  • nice / renice
    • nice -n N cmd : Start a process with nice value N (higher is lower priority).
    • renice N -p PID : Change nice value of an existing PID.
    • ps -o pid,ni,cmd : Verify NI values for running processes.
  • systemctl
    • systemctl enable --now tuned : Enable and start tuned immediately.
