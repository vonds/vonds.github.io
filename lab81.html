Lab 81: Setting Up a Web Server (Apache - HTTP)

« »
Lab Index
  • Scenario
  • Walkthrough
  • Reference

Deploy a basic Apache HTTP web server and serve static content on port 80. Verify service state and confirm delivery end-to-end with a local HTTP request.

services web core

Scenario

You need to deploy a basic Apache HTTP web server to serve a static landing page over port 80 . Your job is to install the package, enable and start the service, place a custom index.html in the web root, then validate the result with a local HTTP request.

Operator context

This is a common “get it online safely” workflow: install, enable, verify status, publish content, then prove it’s reachable with a simple client check before you add TLS, virtual hosts, or firewall rules.

Objective

  • Install Apache (httpd/apache2) using your package manager.
  • Enable and start the web server service with systemd.
  • Publish a custom /var/www/html/index.html .
  • Confirm the server responds locally using curl .

What You’ll Practice

  • Package installation on common distros ( pacman / apt / yum ).
  • Service enablement and immediate start with systemctl enable --now .
  • Writing privileged content safely using tee .
  • Quick HTTP validation using curl against localhost .

Walkthrough

Step 1 : Install the Apache web server package.
Command 
# Pick the command that matches your distro:

# Arch
sudo pacman -S apache

# Debian/Ubuntu
sudo apt install apache2 -y

# RHEL/CentOS/Fedora (traditional)
sudo yum install httpd -y

Installing Apache is the prerequisite for systemd service management and content publishing. The package name varies by distro, but the outcome is the same: Apache binaries and service unit files are placed on the system. 

# Example result (varies by distro):
# httpd/apache2 installed successfully.
Step 2 : Enable and start the Apache service.
Command 
# RHEL-family unit name:
sudo systemctl enable --now httpd

# Debian/Ubuntu unit name:
sudo systemctl enable --now apache2

enable --now ensures Apache starts immediately and also comes back after reboot. If this step fails, check the unit name for your distro and confirm the package installed correctly. 

# Quick verification options:
systemctl status httpd
# or
systemctl status apache2
Step 3 : Create a custom landing page.
Command 
echo 'Hello from LPIC Lab!' | sudo tee /var/www/html/index.html

Writing to /var/www/html typically requires elevated privileges. Using tee is a reliable way to write as root while keeping the command simple and auditable. 

# Confirm file exists and contains the expected content:
sudo cat /var/www/html/index.html
Step 4 : Test the web server locally with curl.
Command 
curl http://localhost

This confirms the full chain: service is listening, request is handled, and content is served from the expected web root. If you don’t get the expected output, verify the service is active and that the file path is correct. 

Hello from LPIC Lab!

Reference

  • pacman -S apache : Installs Apache on Arch-based systems.
  • apt install apache2 -y : Installs Apache on Debian/Ubuntu systems.
  • yum install httpd -y : Installs Apache on RHEL-family systems (package name httpd ).
  • systemctl enable --now <unit> : Enables a service at boot and starts it immediately.
  • /var/www/html : Common default document root for Apache.
  • tee : Writes to privileged paths cleanly when used with sudo .
  • curl http://localhost : Simple HTTP client validation for local service checks.
