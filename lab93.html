Lab 93: Opening Image Files from the Command Line

« »
Lab Index
  • Scenario
  • Walkthrough
  • Reference

Open image files from the CLI using both GUI and terminal-native tools. Validate that assets exist, choose the right viewer for the environment, and confirm a sane “default app” workflow with xdg-open.

cli productivity desktop

Scenario

You have image artifacts (screenshots, diagrams, tickets, docs) and you want to open them directly from the command line. You’ll confirm the files exist, install a lightweight viewer, open images explicitly, and fall back to the system default opener when you don’t want to pick an app.

Operator context

Your approach depends on environment. On a headless server, a GUI viewer won’t help; you’ll either transfer files off box or use a terminal-capable preview tool when available.

Objective

  • Verify image files exist from the shell.
  • Install a lightweight GUI image viewer (feh).
  • Open images directly with the viewer.
  • Optionally install a terminal preview tool (viu).
  • Open images using the system default with xdg-open.

What You’ll Practice

  • Quickly locating image assets using shell globs (*.png, *.jpg).
  • Installing a minimal viewer and launching it from a terminal.
  • Understanding when terminal previews are feasible (TTY vs graphical terminal emulator, font/renderer support).
  • Using xdg-open to hand off to the system’s default application.

Walkthrough

Step 1 : Confirm the image file exists.
Command 
ls *.jpg
# OR
ls *.png
# OR
ls *.jpeg

This is the fastest “is it here?” check. If the glob matches nothing, your shell may print an error or return a non-zero status depending on settings. 

# Expected: one or more filenames, for example:
diagram.png
screenshot-01.jpg
Step 2 : Install a lightweight GUI image viewer.
Command 
sudo apt install feh -y
# OR
sudo dnf install feh -y
# OR
sudo pacman -S feh

feh is a common lightweight viewer for X11 environments. If you don’t have a graphical session, this step won’t be useful. 

# Expected: package installs successfully (or reports already installed).
Step 3 : Open an image with feh.
Command 
feh image.jpg
# OR
feh image.png

Use the explicit filename you found in Step 1. This launches a window and renders the image using your GUI session. 

# Expected: feh opens a window and displays the image.
Step 4 : Install a terminal-based image preview tool (optional).
Command 
sudo apt install viu -y
# OR
sudo pacman -S viu
# OR
cargo install viu

viu renders images as ANSI/Unicode graphics in the terminal. Output quality depends on your terminal and font support. 

# Expected: viu installs successfully (or cargo compiles it).
Step 5 : Preview an image inline in the terminal.
Command 
viu image.jpg
# OR
viu image.png

If you’re in a TTY or a limited terminal, you may get a poor result (or no useful output). In a modern terminal emulator, it typically renders well enough for a quick preview. 

# Expected: a terminal-rendered preview of the image (quality varies by terminal).
Step 6 : Open the image using the system default app.
Command 
xdg-open image.jpg
# OR
xdg-open image.png

xdg-open delegates to whatever your desktop environment has set as the default image viewer. 

# Expected: the image opens in your default GUI application.

Reference

  • ls *.png : List files matching an image extension glob.
  • feh <file> : Lightweight GUI image viewer.
  • viu <file> : Terminal-based image rendering (ANSI/Unicode).
  • xdg-open <file> : Open file with the system default application (desktop environments).
