Lab 89: Managing User Passwords and Aging Policies

« »
Lab Index
  • Scenario
  • Walkthrough
  • Reference

Provision a new user account, set an initial password, and force a password reset at next login to establish a secure first-use workflow. Inspect and harden password aging with chage to align account lifecycle policy with production standards.

users security policy

Scenario

A new user account has been created on a production system. You need to set an initial password, enforce a password change at next login, and configure password aging to meet baseline security requirements.

Operator context

Password and aging settings are both operational and security-critical. You’re enforcing a first-login reset and ensuring the account follows rotation and warning policy so the system doesn’t drift into “never expires” defaults.

Objective

  • Create the user account appuser (if missing).
  • Set an initial password and confirm the password update.
  • Force a password change at next login using passwd -e .
  • Display current password aging settings with chage -l .
  • Apply a stricter aging policy using chage flags for min/max/warn days.

What You’ll Practice

  • Creating users with useradd and ensuring a home directory exists ( -m ).
  • Setting and updating passwords with passwd .
  • Enforcing a password reset at next login with passwd -e .
  • Reading password aging state with chage -l .
  • Applying aging policy controls: -m (min days), -M (max days), -W (warn days).

Walkthrough

Step 1 : Create the user account appuser.
Command 
sudo useradd -m appuser

-m ensures a home directory is created. In production, you typically confirm identity sources (local vs LDAP/SSSD) and naming conventions before provisioning accounts. 

# Expected outcome:
User appuser created.
Step 2 : Set an initial password for appuser.
Command 
sudo passwd appuser

This establishes a usable password so the account can log in and then be forced through a first-login reset. The command prompts interactively and updates the shadow entry for the user. 

Changing password for user appuser.
New password: ********
Retype new password: ********
passwd: password updated successfully
Step 3 : Force a password change at next login.
Command 
sudo passwd -e appuser

Expiring the password immediately ensures the user must choose a new secret at next login. This is a common workflow for onboarding and access restoration. 

passwd: password expiry information changed.
appuser will be forced to reset their password on next login.
Step 4 : Display current password aging configuration.
Command 
sudo chage -l appuser

chage -l prints the account’s password aging state. This is the fast verification step before you change policy so you can compare “before vs after” in a controlled way. 

Last password change                                    : Jan 01, 2025
Password expires                                        : Jan 01, 1970
Password inactive                                       : never
Account expires                                         : never
Minimum number of days between password change          : 0
Maximum number of days between password change          : 99999
Number of days of warning before password expires       : 7
Step 5 : Apply a stricter password aging policy.
Command 
sudo chage -m 7 -M 90 -W 7 appuser

This enforces a minimum of 7 days between changes, a maximum lifetime of 90 days, and a 7-day warning window. After changing policy, re-run chage -l to confirm the new values took effect. 

# Expected outcome:
Password aging policy updated for appuser.

# Verify:
sudo chage -l appuser

Reference

  • useradd -m <user> : Creates a user and ensures a home directory exists.
  • passwd <user> : Sets or updates a user password (interactive prompt).
  • passwd -e <user> : Expires the password immediately, forcing a reset at next login.
  • chage -l <user> : Displays password aging information.
  • chage -m <days> : Sets the minimum days between password changes.
  • chage -M <days> : Sets the maximum password age in days.
  • chage -W <days> : Sets warning days before password expiration.
