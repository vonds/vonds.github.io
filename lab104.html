Lab 104: Installing, Configuring, and Managing OpenVPN

« »
Lab Index
  • Scenario
  • Walkthrough
  • Reference

Build a basic OpenVPN server using TLS certificates generated by Easy-RSA, then validate service state and generate a client certificate for remote access. This lab emphasizes PKI workflow, secure file placement, and systemd service management on a RHEL-style host.

networking security vpn

Scenario

You’re configuring a secure VPN tunnel for remote workers using OpenVPN with TLS-based encryption. Your goal is to stand up a functional OpenVPN server, bootstrap a Certificate Authority (CA), issue server credentials, and confirm the service reaches an “Initialization Sequence Completed” state.

Operator context

VPN is security infrastructure. Treat keys as secrets, apply least privilege to key files, and validate service behavior before distributing client configs.

Objective

  • Install OpenVPN and Easy-RSA.
  • Initialize PKI and build a local CA.
  • Generate and sign a server certificate.
  • Generate Diffie-Hellman parameters.
  • Stage server keys/certs in /etc/openvpn/.
  • Generate an initial server config and start the service.
  • Verify service state and generate a client certificate.

What You’ll Practice

  • Package installation on RHEL-like systems using dnf.
  • Easy-RSA PKI workflow: init-pki, build-ca, gen-req, and sign-req.
  • Server crypto material management and safe placement under /etc/openvpn/.
  • Creating a baseline server config from the upstream sample configuration.
  • systemd lifecycle management: systemctl enable --now and systemctl status.

Step 2 : Set up the PKI directory.
Command 
make-cadir ~/openvpn-ca && cd ~/openvpn-ca
Note

Easy-RSA packaging differs across distros. Some provide make-cadir, others expect you to copy from /usr/share/easy-rsa/. The goal is the same: create a working directory for PKI operations.

The CA is the trust anchor. Protect pki/private/ca.key like a production secret.

Step 6 : Copy certificates and keys into /etc/openvpn/.
Command 
sudo cp pki/ca.crt pki/private/server.key pki/issued/server.crt pki/dh.pem /etc/openvpn/

  cp: copy '/home/lab/openvpn-ca/pki/issued/server.crt' -> '/etc/openvpn/server.crt'
  cp: copy '/home/lab/openvpn-ca/pki/dh.pem' -> '/etc/openvpn/dh.pem'
Security note

Your sample output warns that server.key is group/other accessible. In real deployments, correct file permissions immediately (e.g. chmod 600 /etc/openvpn/server.key and ensure root ownership).

Step 7 : Generate a baseline OpenVPN server configuration.
Command 
sudo gunzip -c /usr/share/doc/openvpn/examples/sample-config-files/server.conf.gz | sudo tee /etc/openvpn/server.conf > /dev/null
  0+1 records in
  0+1 records out
  7556 bytes copied, 0.00231445 s, 3.3 MB/s

This gives you a starting config. In production, you’d explicitly set certificate paths, modern cipher settings, and routing rules, then validate with a controlled client rollout.

Step 8 : Start and enable the OpenVPN service.
Command

At this point you have the CA, server cert/key, and a client cert/key. The next step in a real environment is to generate a client config (.ovpn) and securely distribute client credentials.

Reference

  • dnf install openvpn easy-rsa -y : Install OpenVPN and Easy-RSA tooling.
  • make-cadir : Initializes a working directory for Easy-RSA (packaging dependent).
  • ./easyrsa init-pki : Create PKI structure.
  • ./easyrsa build-ca : Create the CA certificate and private key.
  • ./easyrsa gen-req and ./easyrsa sign-req : Generate and sign certificate requests.
  • ./easyrsa gen-dh : Generate DH parameters.
  • systemctl enable --now openvpn@server : Enable and start the OpenVPN instance for server.conf.
  • systemctl status openvpn@server : Verify service status and check for init success.
